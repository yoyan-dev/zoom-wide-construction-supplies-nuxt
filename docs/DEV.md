Follow my project structure and architecture.

Rules:

- every major page can have a local `_components` folder
- page-specific UI must stay inside `_components`
- pages should mainly compose components
- shared reusable UI should stay in shared components
- stores, composables, utils, and types stay outside page folders
- keep files small, modular, and easy to understand
- avoid crowded page files and logic-heavy components
- inspect the existing structure first
- preserve the current flow unless improvement is requested
- do not modify unrelated files
- always delete unused files

Always output:

- analysis
- folder structure
- component breakdown
- implementation plan
- code changes
- acceptance checklist

Put daily accomplishments in `docs/DAILY-ACCOMPLISMENTS.md`
